Reglas de codificación de vibraciones en resumen
Configuración y configuración de IA

    Prefiera Claude 4.5 o 3.7 thinking para la codificación agente.
    Asegúrese de que las claves de la API estén configuradas correctamente.
    Defina las reglas del proyecto en el IDE para controlar el comportamiento de la IA.

Flujo de trabajo de codificación

    Comience con especificaciones detalladas antes de generar código AI.
    Mantenga las solicitudes de IA pequeñas y precisas para evitar cambios innecesarios.
    Ejecute pruebas con frecuencia para verificar el código generado por la IA.
    Prefiera las pruebas de extremo a extremo sobre las pruebas unitarias para una mejor validación del mundo real.
    Supervise el tamaño del contexto del chat de AI, reinicie las sesiones cuando el rendimiento se descargue.

Código Calidad y Estructura

    Prefiere soluciones sencillas, evita la ingeniería excesiva.
    Elimine la duplicación de código, reutilice las funciones existentes siempre que sea posible.
    Mantenga entornos separados para DEV, TEST y PROD.
    Solo aplique los cambios solicitados, evite modificar las partes no relacionadas.
    No introduzcas nuevas tecnologías o patrones a menos que sea estrictamente necesario.
    Mantenga la base de código limpia y estructurada, refactorie con regularidad.
    Nunca utilice datos simulados en DEV o PROD, solo para pruebas.
    Nunca sobrescriba archivos .env sin aprobación explícita.

Documentación y seguimiento del progreso

    Actualizar continuamente el README (instalación, mantenimiento, información clave).
    Verifique siempre el código generado por la IA antes de continuar.
    Mantenga un archivo TODO (To Do, In Progress, Done) para rastrear el progreso de la IA.

Implementación y control de versiones

    Comprométete con frecuencia para permitir retrocesos fáciles.
    Utilice el visor de IDE incorporado para revertir los cambios cuando sea necesario.
    Ejecute modificaciones de IA en ramas separadas antes de fusionarse con la principal.
    Nunca utilice el modo YOLO en producción, apruebe manualmente implementaciones críticas.

Optimizaciones Adicionales
Apéguese a las pilas tecnológicas que ya tiene el proyecto, en caso de que no hayan apéguese a las populares (Python, JS, SQL, etc.) para obtener un mejor soporte de IA.
Utilice múltiples ventanas de agente de IA para desarrollar características en paralelo.
Aproveche el historial de chat de AI y restaure los puntos de control cuando sea necesario
